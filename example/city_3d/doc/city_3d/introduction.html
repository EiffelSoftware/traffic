Installation of gobo-Documentation

First of all, make sure you have the latest GOBO-Version from the CVS!

To be prepared for the compilation you have to make a geant install in the $GOBO/library/. Then you have to compile the tool gexslt from $GOBO/src/gexslt as known with geant compile.

To use this tool in each directory, you have to add it to the path-variable ($GOBO/src/gexslt/).

You are now prepared to parse the documentation-xml-files. For example go to $EM/doc/sample/ and install this help with geant install. To delete the *.html-files you can use geant clobber.

Create an own Documentation

To start your documentation I recommend to copy and rename the sample-folder from the directory $EM/doc/sample. Please create your folder directly in this directory $EM/doc/! (replace the words "sample" through your foldername in all *.xml-files). It is important to have an index.xml as an overview about the whole documentation.

To add or remove documentation-files, you have also to change the build.geant. At the end of this file, you'll find a list with all activated files ("index" has to be the last item).

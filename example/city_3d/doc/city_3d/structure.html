Structure of gobo-Documentation

The structure for gobo-Documentation for *.xml is as followed: 

<?xml version="1.0"?>
<!--
	description:
	(see current-.xml)
-->
<chapter xmlns="http://www.gobosoft.com/eiffel/gobo/documentation" id="sample/with_code">
<chapterinfo>
	(see current-.xml)
</chapterinfo>
<title>Document Title</title> // Here: Use Gobodoc
<section id="intro"> // A new subchapter
	<title>Introduction</title>
</section>
...
</chapter>

Tag Summary

A little summary for important tags
<para> : paragraph

<programlisting>: mark programcode (see below)

<classname> : mark classnames (example: EM_SURFACE)

<featurename> : mark a featurename (example: my_feature)

<argument> : mark an argument (example: my_argument)

<comment> : mark a comment (example: -- my_comment)

<filename> : mark a comment (example: my_file.bin)

<expression> : mark an expression (see below)

<assign> : mark an assignment (see below) (German: "Zuweisung")

<equal> : mark an equality (see below)

example: 

    set_foo (a_foo: like foo) is
            -- Set foo to a_foo.
        require
            a_foo_not_void: a_foo /= Void
        do
            foo := a_foo
        ensure
            foo_set: foo = a_foo
        end
